# 当前版本待办清单

本文档用于整理 `MyPassword` 当前版本的开发收尾事项，按优先级拆分为：

- 必须做
- 建议做
- 后续做

适用场景：
- 当前版本准备交付测试前
- 当前版本准备正式封版前
- 规划下一个迭代周期时

---

## 一、必须做

以下事项属于当前版本交付前必须完成的内容。

### 1. 主流程真机回归

需要在真机上完整回归以下流程：

- 创建保险库
- 主密码解锁
- 生物识别解锁
- 条目新增
- 条目编辑
- 条目删除
- 条目搜索
- 收藏排序
- 导出备份
- 导入备份
- 修改主密码
- WebDAV 连接测试
- WebDAV 上传同步包
- WebDAV 恢复同步包

### 2. 自动同步第一期验证

重点验证以下场景：

- 主密码解锁进入应用后：
- 新增条目是否触发自动上传
- 编辑条目是否触发自动上传
- 删除条目是否触发自动上传
- 导入备份是否触发自动上传
- 修改主密码是否触发自动上传

异常场景需要验证：

- WebDAV 未配置
- WebDAV 用户名或应用密码错误
- 网络断开
- 远端同步包不存在
- 当前主密码错误
- 备份文件主密码错误

### 3. 升级日志补全

需要检查并补全：

- `docs/项目/升级日志.md`

要求：

- 当前版本的真实更新内容必须完整填写
- 不允许保留“待补充”
- 发布内容要与实际 APK 功能一致

### 4. 手动发版流程固定

每次发版前必须手动执行以下动作：

1. 修改 `pubspec.yaml` 中的版本号
2. 执行 `flutter build apk --release`
3. 将 APK 复制到 `releases/` 目录
4. 按命名规范重命名 APK
5. 更新 `docs/项目/升级日志.md`

### 5. Release 包归档目录启用

要求：

- 创建并启用 `releases/` 目录
- 后续测试包和交付包统一归档到该目录
- 不再只依赖 `build/` 目录中的临时产物

---

## 二、建议做

以下事项不是当前版本必须阻塞交付的内容，但建议尽快补齐，以提升可用性和后续维护效率。

### 1. 增加同步状态展示

建议新增同步状态页或同步状态卡片，至少展示：

- 最近一次本地变更时间
- 最近一次上传时间
- 最近一次下载时间
- 最近一次同步失败原因

### 2. 优化同步入口

建议在设置页增加更明显的同步入口：

- 立即同步一次
- 查看最近同步状态
- 查看当前同步配置状态

### 3. 优化自动同步体验

建议补充以下交互：

- 自动同步进行中提示
- 自动同步成功后的轻量反馈
- 自动同步失败后的持久错误记录

### 4. 优化首页远端更新提示

建议在首页远端更新提示基础上继续增强：

- 增加“去恢复”快捷入口
- 增加“稍后提醒我”
- 增加提示关闭后的再次触发策略

### 5. 生物识别解锁后的同步策略明确化

当前状态：

- 生物识别解锁进入应用后，由于当前会话没有输入主密码，自动上传不会执行

建议明确后续产品策略：

- 保持现状，只允许手动同步
- 或单独设计安全可接受的同步授权机制

### 6. 补基础测试

建议优先补以下测试：

- 备份文件主密码校验
- 导入重复项跳过逻辑
- WebDAV 错误映射逻辑
- 自动同步触发逻辑
